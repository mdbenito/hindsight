Hindsight 0.7.0 is a major release focused on **multilingual search**, **horizontally scaled deployments**, and a **leaner, self-healing graph layer**. It adds three new BM25 backends (ParadeDB pg_search for Citus, PGroonga for CJK/polyglot, and a configurable native language), a new ZeroEntropy embeddings provider, Codex OAuth embeddings, a fully internationalized Control Plane in eight locales, and a major graph-storage cleanup: entity edges are no longer materialized in `memory_links` (~50% fewer rows, ~190 MB freed on a 10k-unit bench), nine unused or redundant indexes are dropped, and a new async link-recompute pipeline keeps the graph correctly capped after deletes. Observation consolidation quality also gets a substantial prompt rewrite, plus a targeted-by-scope variant.

## ParadeDB pg_search for Citus

`HINDSIGHT_API_TEXT_SEARCH_EXTENSION` gains a fourth option: `pg_search`. ParadeDB's pg_search is the only true-BM25 backend that works on a Citus distributed Postgres cluster, which unblocks horizontally scaled deployments that previously had to fall back to the `native` tsvector backend.

The retrieval arm builds the `@@@` predicate via `paradedb.boolean(should => ARRAY[paradedb.match('text', ...), ...])` so multi-field coverage (text + context + text_signals) is preserved without query-string interpolation. A docker-compose recipe ships under `docker/docker-compose/pg_search/` based on the official `paradedb/paradedb:latest-pg17` image.

A new `HINDSIGHT_API_PG_SEARCH_TOKENIZER` knob also lets you pick the tokenizer (`default`, `whitespace`, `raw`, etc.) when the default doesn't fit your corpus.

## Better Multilingual Search

Hindsight has detected the language of incoming content for a while, but BM25 search was effectively English-only — the native tsvector index used English stemming regardless of the bank's actual language, which hurt keyword recall on non-English content and was particularly painful for CJK languages that need bigram-style tokenization to work at all.

0.7.0 fixes that on two fronts:

- The **native BM25 backend** now uses a configurable language dictionary, so a Spanish, French, or German bank gets the right stemming out of the box.
- A new opt-in **PGroonga backend** uses a single polyglot index that handles English, Chinese, Japanese, Korean, and more *simultaneously* — the right choice for mixed-language banks. It ships with a docker-compose recipe.

You can also pin the fact extractor's output language independently of the index language, which is useful when you want to retain in one language and consolidate/recall in another.

See the [Multilingual Support](/developer/multilingual) docs for the full configuration matrix and trade-offs between backends.

## Graph Storage & Maintenance

The graph layer gets three coordinated changes that together cut storage roughly in half and make the graph self-healing after deletes.

**Entity edges are no longer materialized.** Retain used to write a pairwise `link_type='entity'` row to `memory_links` for every pair of units sharing an entity — but recall never actually read those rows. Entity expansion in the retrieval path traverses `unit_entities` directly. On the `recall-perf-medium` bench (10k units), entity rows accounted for **53% of all `memory_links`** — 345k rows and ~190 MB of table + index nobody read, with one unit accumulating 326 outgoing entity links. 0.7.0 stops writing them, derives entity edges on demand for `/graph` and `/stats`, and ships a migration that chunk-deletes the existing rows. The retain pipeline drops the synchronous pairwise loop. API response shapes are unchanged — no SDK regen needed.

**Nine unused or redundant indexes dropped.** A code audit found nine indexes on `memory_links`, `entities`, `documents`, and `unit_entities` that were either dead (no code path exercised them) or fully covered by composite indexes the planner already preferred. They're dropped concurrently in a migration so writers aren't blocked. Combined with the entity-edge cleanup, total graph storage on the same bench is roughly halved.

**Async link recompute fixes under-capped neighbours after deletes.** Deleting a memory unit FK-cascades its incoming temporal/semantic links away, which leaves the *other* end of those edges below cap. Retain only generates links for newly inserted units, so surviving units used to stay permanently under-capped — degrading graph-expansion recall over time. 0.7.0 adds a reactive top-up: inside the delete transaction, `from_unit_ids` that pointed at the doomed units are captured into a new `link_recompute_queue`; after commit, a `link_recompute` task is submitted (deduplicated per bank); the worker drains the queue in batches of 50 and re-runs the same probes retain uses for any unit still below the cap. No change to the hot retain path.

## Control Plane in 8 Languages

The Control Plane UI is now fully internationalized via `next-intl`. Pages move under a `[locale]` segment with locale-prefixed routing (the default English has no prefix), and there's a Globe language switcher in the header. Locales shipped:

- English (`en`) — default, no prefix
- Spanish (`es`), French (`fr`), German (`de`), Portuguese (`pt`)
- Japanese (`ja`), Korean (`ko`), Chinese (`zh`)

The middleware combines i18n routing with the existing auth middleware. Its matcher uses an explicit file-extension allowlist so bank IDs that contain dots (e.g. `SX.Products.GovComply.Build`) still get the correct locale rewrite.

A vitest parity test and a static finder (`scripts/find-untranslated.ts`, exposed as `npm run i18n:check`) walk the TSX AST to flag hardcoded user-facing strings — both wired into CI so future drift fails the build.

## Consolidation Improvements

Observation consolidation gets two complementary changes — one on the control surface, one on quality.

**Targeted re-consolidation.** `POST /v1/default/banks/{bank_id}/consolidate` now accepts an `observation_scopes` parameter so you can re-consolidate only memories matching specific tag combinations, instead of rebuilding the entire bank. A companion `enable_auto_consolidation` flag (configurable per bank) disables the automatic post-retain consolidation entirely, for deployments that prefer to consolidate on a schedule or on-demand.

**Quality.** The consolidation prompt was rewritten to address the root cause of duplicate sibling observations (issue #1566) — the new behaviour strongly prefers updating an existing observation over creating a near-duplicate. A defensive guard also collapses multiple updates to the same observation in a single batch (the LLM occasionally emitted them, and the later write silently overwrote the earlier). Per-bank `observations_mission` now cascades cleanly into the prompt, and a number of reflect-side temporal-reasoning issues surfaced during validation were fixed in the same pass.

## Clear Mental Model

A new `POST /mental-models/{id}/clear` endpoint resets a mental model's content to empty, so the next refresh performs a full re-synthesis regardless of trigger mode. This is useful for periodic compaction of delta-mode models that accumulate drift over many incremental refreshes.

The Control Plane surfaces this in the mental-models detail view, and SDK examples (Python, Node.js, CLI, Go) are included in the docs.

## New Embedding Options

- **ZeroEntropy** is a new embeddings provider. Set `HINDSIGHT_API_EMBEDDINGS_PROVIDER=zeroentropy` and a model.
- **Codex OAuth embeddings**: a new `openai-codex` embeddings provider that uses the existing Codex OAuth token (no separate API key). OpenAI output dimension overrides (`HINDSIGHT_API_EMBEDDINGS_OPENAI_DIMENSIONS`) are also documented for the 384-dimension configuration path.

## Other Notable Changes

- **Alibaba Qwen3 reranker support** in the local cross-encoder path.
- **Right Agent integration** — a new entry in the integrations catalog.
- **OpenCode Go LLM provider** for the `opencode-go` runtime.
- **Ollama Cloud provider** with corrected native-API authentication for cloud endpoints.
- **API access logging env var** to enable Uvicorn access logs without code changes.
- **Per-operation LLM concurrency caps** to keep parallel workloads from overloading a provider.
- **Pre-normalized reranker scores** are now detected and switched to rank-based normalization automatically.
- **Stop sending `temperature`** to Anthropic — the latest models reject it.
- **Webhook batch deliveries** are deduplicated to avoid duplicate retain notifications.
- **Oversized single items in batch retain** are split before the call, so very large payloads no longer fail the whole batch.
- **Disabled reflect tools** are hidden from the agent's system prompt, not just gated at invocation.
- **Invalid embedding dimensions** no longer trigger retries — the response is rejected immediately.
- **User-defined label entities** are no longer subject to fuzzy resolution, so user-defined identities stay stable.
- **Mental model history** is now capped to prevent JSONB overflow, and full refreshes correctly rebase pending delta baselines.
- **Gzip middleware** is enabled so compressed graph responses stay parseable.
- **Control Plane session cookies** are now verified by signature, not by presence — a meaningful auth hardening.
- **Windows embedded UI startup** resolves the absolute `npx` path before spawning, fixing a long-standing launch failure.
- **Tag groups and triggers** are preserved correctly when updating tags.
